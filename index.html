QUICK START

Copy, paste,
investigate

sherlock.yml

Add this file to .github/workflows/ in your repository. Set your OpenAI key as a repo secret and you're done.

Customize the personality, strictness, and dozens of other options to match your team's workflow.

sherlock.yml 
name: SherlockQA Review

on:
  pull_request:
    types: [opened, synchronize, reopened]

permissions:
  contents: read
  pull-requests: write

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: mayurrawte/SherlockQA@v1
        with:
          openai-api-key: ${{ secrets.OPENAI_API_KEY }}
          personality: detective
          review-strictness: balanced

STRICTNESS

Set the bar for
your team

Three levels of scrutiny. Pick one, or change it per-repo.

Lenient

Only flags what truly matters. Ideal for fast-moving teams and small fixes.

  • Critical bugs and crashes
  • Real security vulnerabilities
  • Skips style, tests, minor issues
Approval barLow

Balanced

The default. Catches real problems without nitpicking your semicolons.

  • Null checks, edge cases, logic errors
  • Security vulnerabilities
  • Breaking changes flagged
Approval barMedium

Strict

Thorough analysis for mission-critical code. Nothing gets past.

  • Performance, complexity, race conditions
  • Test coverage, code quality
  • 3–5 QA scenarios per review
Approval barHigh

PRIVACY & SECURITY

Your code stays yours

SherlockQA is transparent about what it reads and where data goes.

Reads diffs only

SherlockQA only reads the PR diff — never your entire codebase, secrets, or environment variables.

Your AI provider

Code is sent to your configured OpenAI or Azure endpoint. SherlockQA never stores or logs any code.

Open source

The entire action is MIT-licensed and open source. Audit every line on GitHub before using it.
